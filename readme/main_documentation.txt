YOUTUBE AUTOMATION PROJECT - MAIN PIPELINE
==========================================
File: main.py

Purpose
-------
main.py is the orchestration layer for the video production pipeline. It does not create assets directly; it calls the scraper, script generator, TTS engine, visual downloader, video assembler, upload-prep step, YouTube uploader, thumbnail generator, and optional Meta cross-posting functions in order.

Entry Points
------------
- python main.py
  Starts the interactive launcher. After a timeout it defaults to automatic mode.
- python main.py --manual
  Runs with manual review. You can let AI pick topics or enter your own topic.
- python main.py --auto
  Runs fully automated.
- python main.py 8_am --auto
  Runs a specific slot automatically.

Supported Slots
---------------
- mid_night
- 4_am
- 8_am
- mid_day
- 4_pm
- 8_pm

If no slot is provided, main.py picks one from the current local hour.

Current Pipeline
----------------
1. Runs YouTube auth pre-flight through auth_check.verify_and_refresh_token().
2. Selects manual or automatic mode.
3. Creates or selects a pending MongoDB task:
   - Manual mode can refine a user topic with AI before saving it.
   - AI topic mode uses NewsScraper to pull RSS stories and select top candidates.
   - Automatic mode selects the first generated topic.
4. Generates and validates a script through ScriptGenerator.
5. Generates scene audio through VoiceEngine.
6. Downloads scene visuals through VisualScout.
7. Assembles FINAL_VIDEO.mp4 through VideoAssembler.
8. Writes FINAL_VIDEO_METADATA.txt through UploadManager.
9. Generates thumbnail.jpg through ThumbnailGenerator.
10. Uploads to YouTube through YouTubeUploader.
11. Optionally cross-posts to Facebook and Instagram through meta_uploader.
12. Appends successful YouTube uploads to production_log.jsonl.

Status Flow
-----------
pending -> scripted -> voiced -> ready_to_assemble -> ready_to_upload -> completed_packaged -> uploaded

Failure Handling
----------------
main.py checks that each stage actually advances the database status. If voice generation, visual download, assembly, or upload prep fails, the task is removed from MongoDB and its generated folder is cleaned up. YouTube upload failures stop cross-posting and final success logging.

Important Files Produced
------------------------
- data/generated_videos_folder/<date>/<slot>/<title>/FINAL_VIDEO.mp4
- data/generated_videos_folder/<date>/<slot>/<title>/FINAL_VIDEO_METADATA.txt
- data/generated_videos_folder/<date>/<slot>/<title>/thumbnail.jpg
- production_log.jsonl
- metadata_<task_id>.txt, created during scripting and removed after successful YouTube upload
