YOUTUBE AUTOMATION PROJECT - MODULE DOCUMENTATION
===============================================
File: main.py

1. What it does?
This file is the "Master Controller" or "Assembly Line Manager". It does not do any of the actual creating itself; instead, it imports all of your core modules (Scraper, Brain, Voice, Visuals, Assembler, Uploader) and runs them in the exact right order. 
Key Features: It includes a "Manual Override" mode that allows you to type in a custom topic instead of using the automated news scraper. It also handles the final cleanup step, deleting temporary text files to keep your computer's hard drive clean, and writes a permanent JSON log of everything created.

2. What are the libraries used?
* sys, os, glob, datetime: Standard system libraries. `os` and `glob` are used at the very end of the script to find and delete all temporary metadata text files (`metadata_*.txt`). `datetime` generates timestamps for the production log.
* asyncio: A library for running asynchronous code. Because `VoiceEngine` uses `edge_tts` (which requires async), `asyncio.run()` is used to safely execute that step within a normal synchronous script.
* argparse: The command-line argument parser. This allows you to run the script with specific instructions from the terminal, such as `python main.py 8_am` or `python main.py --manual`.
* json: Used to append the final success details to `production_log.json`.
* core.*: Imports every single custom class you have built in the `core/` folder.

3. Which is the main function and what does it do?
Main Function: run_creation_pipeline(slot_name, is_manual=False)
This function manages the pipeline sequence:
  1. The Input Branch: If `is_manual` is True, it pauses and asks you to type a topic in the terminal. It loops with the AI up to 10 times to refine your idea before saving it to the database. If `is_manual` is False, it triggers the `NewsScraper` to automatically find a story based on the time slot.
  2. The Assembly Line: It sequentially triggers `Brain.generate_script()`, `Voice.generate_audio()`, `Visuals.download_visuals()`, `Assembler.assemble()`, `UploadManager.prepare_package()`, and finally `YouTubeUploader.upload_video()`. If any step fails, the pipeline halts.
  3. The JSON Logger: After a successful upload, it queries the database for the newest uploaded video and appends its Title, YouTube ID, and Time Slot to a local `production_log.json` file.
  4. The Janitor: It uses `glob` to find any leftover temporary `.txt` files generated by the Brain and deletes them.

4. Helper Functions & Key Variables
* if __name__ == "__main__": The Entry Point. This block only runs if you execute the file directly. It sets up `argparse` to accept the 6 new dynamic time slots (mid_night, 4_am, 8_am, mid_day, 4_pm, 8_pm). 
* Dynamic Fallback Logic: If you run the script without specifying a time (just `python main.py`), it calculates the current hour and automatically assigns the correct 4-hour time block.
