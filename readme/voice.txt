YOUTUBE AUTOMATION PROJECT - VOICE ENGINE
=========================================
File: core/voice.py

Purpose
-------
VoiceEngine converts scripted scenes into MP3 narration files with Edge TTS. It processes scenes concurrently and stores audio paths and durations back into MongoDB.

Main Class
----------
VoiceEngine

Input
-----
The engine looks for one MongoDB task with status:

scripted

Required task fields:
- script_data
- folder_path
- voice_model

Current Behavior
----------------
- Creates the task folder if it does not exist.
- Generates one MP3 file per scene:
  voice_0.mp3, voice_1.mp3, etc.
- Uses the task's voice_model, falling back to en-US-GuyNeural if needed.
- Retries each voice generation up to 3 times.
- Rejects missing, tiny, or shorter-than-0.5-second audio files.
- Limits concurrent TTS calls with MAX_CONCURRENT_TTS.

Environment Variable
--------------------
- MAX_CONCURRENT_TTS
  Optional. Defaults to 3.

Key Methods
-----------
- _generate_single_segment(text, voice, path, rate="+10%")
  Creates and validates one MP3.
- _process_scene_task(i, scene, folder, selected_voice)
  Worker for one scene.
- generate_audio()
  Runs all scene audio jobs and updates the task.

Output
------
Each scene receives:
- audio_path
- duration

On full success, task status changes to:

voiced

Failure Handling
----------------
If any scene fails, the method prints failed scene details and does not advance the task status. main.py then treats the stage as failed.
