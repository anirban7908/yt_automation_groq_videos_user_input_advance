YOUTUBE AUTOMATION PROJECT - MODULE DOCUMENTATION
===============================================
File: core/voice.py

1. What it does?
This file is the "Narrator" and "Timekeeper" of your automation pipeline. Its primary job is to take the text script generated by the Brain and convert it into a high-quality, human-like voiceover using Microsoft Edge's Neural TTS engine.
Key Upgrade: This file is now completely dynamic. Instead of hardcoding a single voice (like "en-US-GuyNeural") for every video, it reads the designated voice model from the database (e.g., a British documentary voice for Space, or a suspenseful voice for True Crime). It also defers completely to the AI in `brain.py` for visual pacing, ensuring the exact number of images the AI requested are downloaded.

2. What are the libraries used?
* edge_tts: A Python library that interfaces with Microsoft Edge's online Text-to-Speech service. It provides access to high-quality, realistic neural voices for free and allows for speed adjustments (e.g., `rate="+10%"`).
* mutagen (specifically MP3): A library for handling audio metadata. After generating the audio file, the code needs to know exactly how long it is (down to the millisecond) to plan the visual cuts. `MP3(path).info.length` provides this precise duration.
* os: Standard Python library for file system operations. Used to construct file paths (`os.path.join`) so the audio files are saved correctly inside the specific video's folder.
* core.db_manager.DBManager: Your custom class that handles database connections. It fetches tasks with the status "scripted" and, after generating audio and logging timing, updates the task to "voiced".

3. Which is the main function and what does it do?
Main Function: `generate_audio(self)`
This function manages the transformation from text to sound:
  1. Fetching: Asks the database for a task where status is "scripted".
  2. Voice Selection: Reads the `voice_model` assigned to this specific video from the database.
  3. Looping & Generating: Iterates through every scene in the script. Calls `edge_tts` using the selected voice and saves it as an MP3 file (e.g., `voice_0.mp3`).
  4. Measuring: Uses `mutagen` to check the exact length of the generated audio file.
  5. Syncing: It reads the `image_count` that was determined dynamically by the AI in `brain.py` (1, 2, or 3 images) and calculates how long each image should stay on screen (`img_duration = duration / img_count`).
  6. Saving: Updates the script data in the database with the audio path and precise timing, then marks the status as "voiced".

4. Helper Functions & Key Variables
* selected_voice = task.get("voice_model", "en-US-GuyNeural"): This grabs the specific Microsoft Edge voice code (like "en-GB-RyanNeural") that was assigned by the scraper, defaulting to "Guy" if none is found.
* communicate = edge_tts.Communicate(text, selected_voice, rate="+10%"): The voice generation command. It applies the dynamic voice and a 10% speed boost to make the narration sound more energetic for YouTube Shorts.
* The Pacing Handoff: Previously, this file used math (`math.ceil`) to guess how many images a scene needed. That logic was removed. Now, it strictly trusts the `scene.get("image_count")` generated by the Meta-Prompting brain, ensuring perfect alignment between the AI's visual keywords and the final video assembly.
