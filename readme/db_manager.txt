YOUTUBE AUTOMATION PROJECT - DATABASE MANAGER
=============================================
File: core/db_manager.py

Purpose
-------
DBManager centralizes MongoDB access, task creation, duplicate checks, and generated-video folder creation.

Main Class
----------
DBManager

Environment Variables
---------------------
- MONGO_URI
  Required.
- DB_NAME
  Optional. Defaults to yt_automation.
- GENERATED_VIDEOS_DIR
  Optional. Defaults to data/generated_videos_folder.

Database Target
---------------
DBManager connects to:

collection: video_tasks_gork

within the configured database.

Key Methods
-----------
- sanitize_filename(name)
  Removes unsafe filename characters and normalizes spaces/dashes.
- get_video_folder(slot, title)
  Creates and returns a generated-video folder for the current date, slot, and title.
- get_used_niches_today()
  Returns niches already used today to help the scraper rotate topics.
- task_exists(new_title, source_url=None)
  Checks recent duplicate source URLs and near-duplicate titles from the last 7 days.
- add_task(title, content, source="manual", status="pending", extra_data=None)
  Creates a task document and inserts it into MongoDB.

Task Fields Created by add_task()
---------------------------------
- title
- content
- source
- status
- source_url
- niche
- slot
- folder_path
- pexels_style
- voice_model
- target_language
- hashtags
- created_at

Folder Layout
-------------
Generated content is stored under:

data/generated_videos_folder/<date>/<slot>/<safe_title>/

or the path configured by GENERATED_VIDEOS_DIR.
