YOUTUBE AUTOMATION PROJECT - SETUP AND RUN GUIDE
================================================

Project Summary
---------------
This project creates and uploads short-form YouTube videos from RSS/news or manual topics. The pipeline uses Groq for script/metadata generation, Edge TTS for narration, multiple media sources for visuals, MoviePy/Whisper for assembly, MongoDB for task state, YouTube Data API for upload, and optional Meta Graph API calls for Facebook/Instagram cross-posting.

Install
-------
1. Create and activate a virtual environment.
2. Install Python dependencies:

pip install -r requirements.txt

3. Install CLIP separately if you want AI-assisted video start selection:

pip install git+https://github.com/openai/CLIP.git

Required Configuration
----------------------
Create a .env file in the project root. At minimum, configure:

MONGO_URI=...
GROQ_API_KEY=...

Common optional keys:

DB_NAME=yt_automation
GENERATED_VIDEOS_DIR=data/generated_videos_folder
PEXELS_API_KEY=...
PIXABAY_API_KEY=...
UNSPLASH_ACCESS_KEY=...
POLLINATIONS_API_KEY=...
MAX_CONCURRENT_TTS=3
MAX_DOWNLOAD_SIZE_MB=250
YOUTUBE_PRIVACY_STATUS=private
YOUTUBE_CLIENT_SECRETS_FILE=client_secrets.json
YOUTUBE_TOKEN_FILE=token.pickle
META_ACCESS_TOKEN=...
FB_PAGE_ID=...
IG_USER_ID=...

Google/YouTube One-Time Setup
-----------------------------
1. Open Google Cloud Console.
2. Create or select a project.
3. Enable YouTube Data API v3.
4. Configure the OAuth consent screen.
5. Add your YouTube account as a test user if the app is still in testing.
6. Create an OAuth Client ID with application type Desktop App.
7. Download the credentials JSON.
8. Save it as client_secrets.json in the project root, or set YOUTUBE_CLIENT_SECRETS_FILE.
9. Run auth_check.py or main.py once and complete the browser authentication.
10. token.pickle will be created and reused by later runs.

Run Commands
------------
- python main.py
  Interactive launcher with timed default to automatic mode.
- python main.py --manual
  Manual review mode.
- python main.py --auto
  Fully automated pipeline.
- python main.py mid_day --auto
  Automated pipeline for a specific slot.
- python scheduler.py
  Runs the six daily scheduled slots.
- python auth_check.py
  Verifies or refreshes YouTube OAuth credentials.

Pipeline Statuses
-----------------
pending -> scripted -> voiced -> ready_to_assemble -> ready_to_upload -> completed_packaged -> uploaded

Generated Output
----------------
Successful runs produce files under:

data/generated_videos_folder/<date>/<slot>/<title>/

Key outputs:
- FINAL_VIDEO.mp4
- FINAL_VIDEO_METADATA.txt
- thumbnail.jpg

Operational Notes
-----------------
- production_log.jsonl records successful YouTube uploads.
- logs/ contains scheduler and per-run logs.
- metadata_<task_id>.txt files in the project root are temporary script metadata files and should disappear after a successful upload.
- Instagram uploads require a public video URL in task.public_url. Local video files cannot be uploaded directly to Instagram through this API path.
