YOUTUBE AUTOMATION PROJECT - SCHEDULER
======================================
File: scheduler.py

Purpose
-------
scheduler.py runs main.py automatically at configured times. It is intended for hands-free operation and keeps each run isolated in its own log file.

Schedule
--------
The active schedule is defined in SCHEDULE_CONFIG:
- mid_night: 00:00
- 4_am: 04:00
- 8_am: 08:00
- mid_day: 12:00
- 4_pm: 16:00
- 8_pm: 20:00

How It Runs Jobs
----------------
Each scheduled job launches:

python main.py <slot> --auto

The scheduler streams each run's output to:

logs/run_<slot>_<timestamp>.log

It also writes scheduler-level logs to:

logs/scheduler.log

Concurrency Protection
----------------------
Only one pipeline run is allowed at a time. If a slot fires while another slot is still running, the newer job is skipped and recorded in job history.

Timeout Protection
------------------
JOB_TIMEOUT is currently 3.5 hours. If a pipeline run exceeds that limit, scheduler.py kills the process tree so Python and FFmpeg/MoviePy child processes do not keep running in the background.

Missed Job Recovery
-------------------
On startup, scheduler.py checks logs/job_history.json. If a configured slot has already passed today and has not recorded a success or failure, the scheduler starts the most recent missed slot immediately.

Auth Pre-Flight
---------------
Before scheduling anything, scheduler.py calls auth_check.verify_and_refresh_token(). If YouTube credentials are missing, expired without refresh support, or invalid, the scheduler exits instead of starting jobs that cannot upload.

Generated Runtime Files
-----------------------
- logs/scheduler.log
- logs/job_history.json
- logs/run_<slot>_<timestamp>.log

Common Commands
---------------
- python scheduler.py
  Start the scheduler.
- Ctrl+C
  Stop the scheduler gracefully.
