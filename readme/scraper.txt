YOUTUBE AUTOMATION PROJECT - MODULE DOCUMENTATION
===============================================
File: core/scraper.py

1. What it does?
This file is the "Content Hunter" of your automation pipeline. Its job is to go out to the internet, find fresh news articles based on the current 4-hour time block (e.g., 8_am, mid_day), and save them into your database.
Key Upgrade: Instead of just picking a random story, it uses the Groq API (Llama 3.3) to analyze all found headlines and select the TOP 3 with the highest potential to go viral. It then filters those against your database to ensure you never make a duplicate video. If all 3 have been used, it loops back and tries again automatically.

2. What are the libraries used?
* requests: The standard Python library for sending HTTP requests. Used to download the raw XML data from news websites.
* feedparser: A specialized library that takes the messy XML from RSS feeds and parses it into clean, readable Python lists (extracting titles, summaries, and links).
* random: Used as a fallback safety net. If the AI fails to pick a winner, `random.choice` takes over to ensure the pipeline never stalls.
* datetime: Used to check the current hour so the bot knows which of the 6 time slots to trigger (e.g., "4_pm").
* groq: The official Groq API client. Replaces local models to provide lightning-fast, cloud-based AI reasoning for judging the most viral headlines.
* json & re: Used to parse and clean the strict JSON outputs returned by the Groq AI, ensuring the bot can accurately extract the selected headline numbers.
* os & dotenv: Used to securely load your hidden API keys (GROQ_API_KEY) from your local .env file.
* core.db_manager.DBManager: Your custom database class. Used to save new tasks and check history to prevent duplicate videos.

3. Which is the main function and what does it do?
Main Function: `scrape_targeted_niche(self, forced_slot=None)`
This orchestrates the entire gathering process:
  1. Strategy Selection: Identifies the current time slot and randomly selects a niche from `MASTER_NICHES` that hasn't been used yet today.
  2. Gathering: Loops through every RSS feed for that niche and compiles a list of candidates.
  3. The Judge (AI): Calls `pick_top_3_viral_topics` to let Groq analyze the candidates and return the 3 most viral options.
  4. The Filter Loop: Checks the top 3 against the database. It loops up to 4 times to guarantee it finds a completely unique, unused viral story.
  5. Saving: Commits the final winner to the database with the status "pending", attaching specific hashtags and assigning the correct voice model.

4. Helper Functions & Key Variables
* MASTER_NICHES: The master configuration dictionary. Replaces the old `niche_map`. It holds the specific RSS feeds, custom hashtags, and designated Microsoft Edge-TTS voice models (e.g., British narrator for Space) for 7 different topics.
* pick_top_3_viral_topics: Sends the headlines to Groq with strict instructions to return a JSON array of the 3 best indices. 
* get_time_slot: Calculates the current hour and returns one of the 6 new dynamic slots (mid_night, 4_am, 8_am, mid_day, 4_pm, 8_pm).
* fetch_rss: Reaches out to the website pretending to be a real web browser (using a Mozilla User-Agent header) to bypass bot-blockers, returning the top 10 articles from that feed.
