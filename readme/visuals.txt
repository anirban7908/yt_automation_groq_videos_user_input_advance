YOUTUBE AUTOMATION PROJECT - VISUAL SCOUT
=========================================
File: core/visuals.py

Purpose
-------
VisualScout downloads scene visuals for voiced tasks. It uses scene keywords, trigger words, niche context, source routing, AI-assisted candidate ranking, and fallback searches to collect the media used by the assembler.

Main Class
----------
VisualScout

Input
-----
The scout looks for one MongoDB task with status:

voiced

Required task fields:
- script_data
- folder_path
- niche

Supported Sources
-----------------
- Pexels video
- Pixabay video
- Unsplash image
- Pixabay image
- Pexels image
- NASA media search
- Wikimedia Commons
- Internet Archive

Important Environment Variables
-------------------------------
- PEXELS_API_KEY
- PIXABAY_API_KEY
- UNSPLASH_ACCESS_KEY
- MAX_DOWNLOAD_SIZE_MB, default 250

Source Routing
--------------
VisualScout picks source order by niche. Space or astronomy keywords can trigger a NASA-first route. Other niches prefer different mixes of stock video, archive material, Wikimedia, or images depending on the subject.

Fallback Strategy
-----------------
For each visual request:
1. Build a contextual query from the keyword, scene text, all scene keywords, and niche context.
2. Try the niche-specific source order.
3. Try alternate keywords from the same scene.
4. Try a simplified noun from the keyword.
5. Try Wikimedia as a final real-media source.
6. Create a dark 1080x1920 placeholder image only if every source fails.

Duplication Avoidance
---------------------
The module tracks used source IDs per run. It avoids reusing Pexels, Pixabay, NASA, Wikimedia, Internet Archive, and generic source identifiers in the same video.

Key Methods
-----------
- _route_source_order(keyword, niche)
  Chooses the source waterfall.
- _build_context_query(keyword, niche, scene_text, all_keywords)
  Builds a richer search query.
- _rank_candidates(...)
  Scores source candidates against keyword, scene, and niche terms.
- _ai_choose_best_visual(...)
  Uses AIEngine to choose from ranked candidates when available.
- _fetch_single_visual(...)
  Runs source search and fallback logic for one requested visual.
- _download_visuals_async(...)
  Fetches visuals concurrently.
- download_visuals()
  Synchronous wrapper used by main.py.

Output
------
Each scene receives an image_paths list containing local media paths.

On success, MongoDB is updated to:

ready_to_assemble
