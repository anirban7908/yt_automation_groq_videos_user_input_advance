YOUTUBE AUTOMATION PROJECT - MODULE DOCUMENTATION
===============================================
File: core/visuals.py

1. What it does?
This file is the "Cinematographer" and "Location Scout" of your automation pipeline. Its job is to visualize the script. It reads the unique keywords generated by the Meta-Prompting Brain (e.g., "Cyberpunk hacker typing fast"), searches the internet for high-quality B-roll, downloads the files, and assigns them to the specific scenes in your video.
Key Upgrade: This file no longer relies primarily on static stock photos. It now actively hunts for high-definition MP4 videos using the Pexels Video API to create highly engaging, fast-paced Shorts. It also features a specific "Hero Hook" mechanism that uses a custom Google Image Scraper to find highly relevant, specific images for the very first frame of the video to maximize Click-Through Rate.

2. What are the libraries used?
* os & time: Standard Python libraries. `os` is used to build safe file paths to save the downloaded media. `time` is used to pause the script briefly (`time.sleep`) between downloads to prevent getting banned by the APIs for requesting files too quickly.
* requests: The core tool for talking to external APIs. It sends the search queries to Pexels/Unsplash/Google and downloads the actual binary video/image data.
* random: Used to pick a random result from the API responses so the videos don't look repetitive if the same keyword is ever used twice.
* re (Regular Expressions): Used specifically in the `search_google_images` function to scan the raw HTML of a Google Search page and extract the hidden direct URLs to the image files.
* core.db_manager.DBManager: Your custom database class. It fetches tasks with the status "voiced" and updates them to "ready_to_assemble" once all media is secured.
* dotenv: Securely loads your API keys (UNSPLASH_ACCESS_KEY, PEXELS_API_KEY) from your local .env file.
* PIL (Pillow) & io: Python Imaging Library. It opens downloaded image bytes directly in memory (`io.BytesIO`) to verify the file is not corrupted before saving it. It is also used to generate a blank black canvas (`Image.new`) as an absolute last resort if all downloads fail.

3. Which is the main function and what does it do?
Main Function: `download_visuals(self)`
This is the director that manages the visual gathering process:
  1. Fetching: Asks the database for a task where status is "voiced".
  2. Planning: Loops through every scene and checks exactly how many visuals the Brain requested (`image_count`) and what the `keywords` are.
  3. The Waterfall Hunt: For every requested visual, it executes a strict fallback sequence:
     a. If it's the very first visual (Scene 0, Image 0), it forces a Google Image Search to find a highly specific "Hero" image.
     b. If not, it searches the Pexels Video API for an MP4.
     c. If the video fails, it falls back to Unsplash/Pexels for a static JPG.
     d. If the specific keyword fails entirely, it tries searching using the *other* keywords generated for that scene.
     e. Final Safety Net: If the internet completely fails, it draws a black placeholder image so the video editor doesn't crash.
  4. Saving: Updates the database with the local file paths of all downloaded media and changes the status to "ready_to_assemble".

4. Helper Functions & Key Variables
* use_pexels_video_search: The new core video engine. It searches the Pexels API, automatically appends the word "aesthetic" to the query to force higher-quality results, filters for MP4 files, sorts them by the highest resolution available, and downloads the best one.
* search_google_images: A custom web scraper. It disguises itself as a Chrome browser (using headers) to search Google Images. It uses Regex to rip the direct `.jpg` links straight from the search results page, bypassing the need for a paid Google API.
* is_valid_image: Quality Control. Tries to open downloaded image bytes with Pillow. If it's a corrupted file or a 404 error page, it returns False so the script knows to try another file.
