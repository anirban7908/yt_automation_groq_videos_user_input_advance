YOUTUBE AUTOMATION PROJECT - SCRIPT GENERATOR
=============================================
File: core/brain.py

Purpose
-------
ScriptGenerator turns a pending task into a validated Shorts package: narration scenes, visual search keywords, trigger words, title, description, hashtags, and tags.

Main Class
----------
ScriptGenerator

Current Generation Flow
-----------------------
1. Reads the next pending task or receives a task from main.py.
2. Builds a niche-specific expert role.
3. Uses AIEngine to create a custom persona and core subject.
4. Generates a 6-8 scene English narration for a 90-110 second Short.
5. Generates packaging JSON with scene keywords, trigger words, image counts, title, description, hashtags, and tags.
6. Validates the JSON structure and visual keyword quality.
7. Cleans scene text so TTS does not read markdown or scene numbering.
8. Saves approved script data to MongoDB and marks the task scripted.

Key Methods
-----------
- get_expert_role(niche)
  Maps each niche to a suitable writing persona.
- repair_json(json_str)
  Attempts to strip non-JSON wrapper text from a model response.
- validate_script(data)
  Enforces 6-8 scenes, required text, keywords, trigger_words, and image_count.
- generate_narration(...)
  Creates the spoken scene list.
- generate_packaging(...)
  Creates metadata and visual planning JSON.
- generate_meta_prompt(...)
  Creates a custom system prompt and core subject.
- generate_script_for_task(task)
  Used by manual review mode; returns data without saving.
- regenerate_with_feedback(task, feedback)
  Re-runs generation with user notes.
- approve_and_save(task, data)
  Saves a user-approved script.
- generate_script()
  Automatic-mode method that processes the next pending task.

Validation Rules
----------------
- Scene count must be 6 to 8.
- Every scene needs text, keywords, trigger_words, and image_count.
- image_count must be 1, 2, 3, or 4.
- keywords and trigger_words counts must match image_count exactly.
- Duplicate or non-filmable visual terms are discouraged or rejected.
- Banned exact keywords include terms such as abstract, concept, icon, background, texture, propaganda, and certain unsafe/person-only terms.
- Banned contained terms include animated, animation, infographic, diagram, 3d, illustration, visualization, and render.

Output
------
On success, MongoDB is updated with:
- script_data
- title
- ai_description
- ai_hashtags
- ai_tags
- status: scripted

It also creates a temporary root metadata file:

metadata_<task_id>.txt

That root metadata file is removed after successful YouTube upload.
