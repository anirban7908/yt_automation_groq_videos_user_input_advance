YOUTUBE AUTOMATION PROJECT - YOUTUBE UPLOADER
=============================================
File: core/uploader.py

Purpose
-------
YouTubeUploader uploads packaged videos to YouTube, applies generated metadata, optionally uploads a generated thumbnail, and marks the task uploaded.

Main Class
----------
YouTubeUploader

Input
-----
The uploader looks for the oldest MongoDB task with status:

completed_packaged

Required task fields:
- final_video_path
- title
- ai_description
- ai_hashtags
- ai_tags
- niche
- source_url
- folder_path

Auth Files
----------
Defaults:
- client_secrets.json in the project root
- token.pickle in the project root

Environment overrides:
- YOUTUBE_CLIENT_SECRETS_FILE
- YOUTUBE_TOKEN_FILE

Other Environment Variables
---------------------------
- YOUTUBE_PRIVACY_STATUS
  Optional. Defaults to private.

Upload Behavior
---------------
- Builds a YouTube Data API v3 client.
- Refreshes or creates OAuth credentials if needed.
- Uses resumable uploads in 4 MB chunks.
- Retries recoverable upload interruptions up to 10 times.
- Treats HTTP 400, 401, 403, and 404 as fatal upload API errors.
- Adds #Shorts when generated hashtags do not already include it.
- Trims tags to fit YouTube's tag length constraints.
- Uploads thumbnail.jpg from the task folder if present.

Category Mapping
----------------
The uploader maps niches to YouTube category IDs. Examples:
- space and tech_ai -> Science & Technology
- war_news -> News & Politics
- animals_nature -> Pets & Animals
- general -> Entertainment

Output
------
On success, MongoDB is updated with:
- status: uploaded
- youtube_id
- uploaded_at

Cleanup
-------
After successful upload, the temporary root metadata_<task_id>.txt file is deleted if it exists.
